# Roadmap

## 当前阶段

项目已经完成从“可跑通”到“可评测”的第一阶段建设：

- 文档入库链路可用
- 检索链路可用
- 外部 LLM 问答可用
- 单文档、多文档、行业样本评测可用

当前主问题已经从“有没有能力”转为：

- 如何继续提升答案质量
- 如何降低外部 LLM 延迟
- 如何让评测和迭代更稳定

## P0

### 1. 问答路由

目标：
- 简单事实题优先走 fallback 摘要
- 复杂综合题再调用外部 LLM

价值：
- 明显降低平均延迟
- 降低模型成本
- 提升产品可用性

### 2. 上下文压缩继续优化

目标：
- 进一步缩短传给 LLM 的上下文
- 保留最相关句子和实体词

价值：
- 提升 `ROUGE-L F1`
- 降低生成时间
- 减少模型跑偏

### 3. 端到端评测分层执行

目标：
- fallback 全量回归
- 外部 LLM 抽样回归
- 关键版本再跑更大样本

价值：
- 缩短评测时间
- 保持回归频率
- 降低模型波动对迭代节奏的影响

## P1

### 4. 更强 rerank

目标：
- 接入更高质量的 rerank 模型或服务

价值：
- 提升引用精度
- 提升上下文质量

### 5. 更强引用组织

目标：
- 控制重复引用
- 减少同义重复片段
- 提高 `Section Precision@K`

价值：
- 让答案更聚焦
- 降低上下文噪声

### 6. 评测趋势看板

目标：
- 保存多轮评测结果
- 做横向对比和趋势追踪

价值：
- 清楚看到每次优化的真实收益
- 方便版本管理

## P2

### 7. 异步化

目标：
- 文档处理改成后台任务
- 大问答支持异步或流式输出

价值：
- 更适合大文件和长回答
- 提升前端体验

### 8. 多知识库 / 多空间

目标：
- 从单一文档集合扩到知识空间管理

价值：
- 更接近真实企业知识库场景

### 9. 观测与告警

目标：
- 记录 embedding 失败率
- 记录 LLM 超时率
- 记录回退比例

价值：
- 更快发现线上真实问题

## 里程碑建议

### M1

- 完成问答路由
- 完成第二轮上下文压缩
- 把 `deepseek-chat` 设为默认生成模型策略

### M2

- 完成 rerank 增强
- 完成引用组织优化
- 完成评测趋势报表

### M3

- 完成异步问答
- 完成多知识空间基础能力

## 当前建议优先级

如果只做一件事，优先做：

1. 问答路由

如果连续做三件事，建议顺序：

1. 问答路由
2. 上下文压缩继续优化
3. 外部 LLM 抽样评测自动化
