RAG 过程可视化

这块适合你做分享时直接讲流程。左边是文档入库链路，右边是问答链路，展示了当前系统如何把原始文档转成可检索、可引用、可回答的知识库。

文档入库链路

把原始文件变成可检索的知识块和向量。

Offline Indexing
01
上传原始文档
支持 `txt / md / pdf / docx`，文件先保存到本地存储，再写入文档元数据。
storage/uploads document
↓
02
解析与清洗
提取正文、识别页码、清掉换行断词和噪声，让后续切分更稳定。
file_parser text_cleaner
↓
03
切分 Chunk
按段落、章节和页范围切成知识块，并保留 `section_title / page_start / page_end`。
document_chunk page range section title
↓
04
生成向量并入库
把每个 chunk 送到 embedding 模型生成向量，写入 PostgreSQL / pgvector，形成可检索知识索引。
Embedding pgvector Redis Worker
分享时可以把这一侧理解成：“把长文档加工成结构化知识块和向量索引”，后续问答就是在这些 chunk 上做召回和生成。

问答链路

把用户问题转成检索、排序和最终答案。

Online Retrieval + Generation
01
理解并改写问题
先做 Query Rewrite，把口语化问题改成更适合检索的表达。
query rewrite question profiling
↓
02
混合召回候选片段
同时做向量检索、PostgreSQL 全文检索和关键词补充，合并出候选证据。
pgvector fulltext keyword
↓
03
Rerank 与去重
利用本地 rerank 和结构化字段，把最相关的章节排到前面，并合并重复证据。
local rerank section title semantic tags
↓
04
LLM 生成答案并返回引用
把精排后的 top chunks 交给 LLM 组织回答，同时返回引用编号和来源片段。
DeepSeek citation traceable answer
分享时可以把这一侧理解成：“先找证据，再让模型基于证据说话”，这样能降低幻觉并让答案可追溯。
